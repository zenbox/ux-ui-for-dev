Farben und Typografie in Ionic

Diese Seite zeigt, wie Ionic Farben ueber CSS-Variablen und Typografie ueber globale und lokale Text-Tokens steuert.

1) Farb-Tokens in :root Corporate + Semantisch
Primary --ion-color-primary
Secondary --ion-color-secondary
Tertiary --ion-color-tertiary
Success --ion-color-success
Warning --ion-color-warning
Danger --ion-color-danger

Komponenten verwenden diese Variablen automatisch, sobald du z. B. color="primary" oder color="danger" setzt.

 2) Typografie Globale Schriftfamilie + lokale Skala
Text XS: Hilfstext, Metadaten
Text SM: Labels und Nebeninfos
Text MD: Standard-Fliesstext in Inhalten
Text LG: Intro-Text und hervorgehobene Inhalte
Text XL: Abschnittsueberschrift

Die Basis kommt von --ion-font-family. Die eigentliche Skala steuerst du in deinen Klassen, Utility-Tokens oder Component-Styles.

Color API

  1. Design-Token in :root definieren: --ion-color-primary usw.
  2. Fuer jede Farbe auch -contrast, -shade und -tint pflegen.
  3. In Ionic-Komponenten nur noch color="..." verwenden.
:root { --ion-color-primary: #1565c0; --ion-color-primary-contrast: #ffffff; --ion-color-primary-shade: #1259a9; --ion-color-primary-tint: #2d74c6; } Speichern

Type API

  1. Globale Schriftfamilie ueber --ion-font-family setzen.
  2. Eine projektweite Textskala als Klassen/Tokens definieren (z. B. .t-sm, .t-md, .t-xl).
  3. Komponenten-Inhalt konsequent mit dieser Skala stylen, damit die Typografie konsistent bleibt.
:root { --ion-font-family: "Barlow", "Avenir Next", "Segoe UI", sans-serif; } .t-md { font-size: 1rem; line-height: 1.5; } .t-xl { font-size: 1.375rem; line-height: 1.3; font-weight: 700; }
